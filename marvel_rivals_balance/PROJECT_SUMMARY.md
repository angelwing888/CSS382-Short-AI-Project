## How It Works

### Data Flow
```
Marvel Rivals Website
        ↓
    [Scraper] (scraper.py)
        ↓
  balance_data.json
        ↓
   [Flask API] (app.py)
        ↓
  [Web Browser] (index.html)
        ↓
      User
```

### Key Components

**1. Scraper** (`scraper.py`)
- Fetches Marvel Rivals official balance post API
- Parses each patch for character adjustments
- Saves to `balance_data.json`
- Run: `python scraper.py`

**2. Web Server** (`app.py`)
- Flask backend serving API
- Routes:
  - `/` - Main page
  - `/api/heroes` - List all heroes
  - `/api/hero/<name>` - Get hero details
  - `/api/stats` - Overall statistics

**3. Frontend** (`templates/index.html`)
- Modern responsive UI
- Real-time search
- Click to expand hero adjustments
- Shows patch, date, and changes

## API Reference

All endpoints return JSON.

### GET /api/heroes
List all heroes with adjustment counts.
```json
[
  {"name": "Spider-Man", "adjustment_count": 5},
  {"name": "Iron Man", "adjustment_count": 3}
]
```

### GET /api/hero/<name>
Get adjustments for specific hero.
```json
{
  "name": "Spider-Man",
  "adjustments": [
    {
      "patch": "Marvel Rivals Version 20260326",
      "date": "2026-03-26",
      "adjustment": "Increased web swing speed by 10%"
    }
  ]
}
```

### GET /api/stats
Overall statistics.
```json
{
  "total_heroes": 42,
  "total_adjustments": 156,
  "most_adjusted": "Spider-Man"
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `ModuleNotFoundError: flask` | Run: `pip install -r requirements.txt` |
| `balance_data.json not found` | Run: `python scraper.py` |
| Port 5000 in use | Change port in app.py or kill process |
| Scraper timeout | Check internet, try again (takes 2-5 min) |
| Data not updating | Run scraper manually or check scheduled job |
